### 📝 主编说

今天大家都在讨论“AI 代理”和“视频生成”，但表面热闹之下，真正的信号藏在细节里。

**表面信号：** 大家都在聊 `Oak`，一个为 AI 代理设计的 Git 替代品（129 条评论），以及 `html-video`，一个让 AI 生成视频的开源项目（3446 star）。这两个话题吸引了大量注意力，但都是“大饼”——前者需要替换开发者的基础设施，后者需要与 OpenAI Sora 等巨头竞争。

**真正可构建的信号：** 是 `Recall`（Local project memory for Claude Code，81 条评论）和 `PMB`（local-first memory for AI coding agents over MCP，28 分）。这两个项目指向同一个痛点的不同解法：**AI 代理没有长期记忆**。开发者们在 HN 上抱怨“Claude Code 每次都要重新理解我的项目”，这就是一个明确的、能行动的信号。

**谁会先付钱？** 已经付费使用 Claude Code、Cursor 或 Copilot 的 **独立开发者和小团队**。他们每月为 AI 工具支付 $20-50，但忍受着“每次对话都像第一次见面”的低效。他们愿意为“让代理记住上次改到哪了”这种功能付 $9-29/月。

**为什么是这周？** 因为 `Recall` 和 `PMB` 同时爆发，说明这个需求已经从“个别人的抱怨”变成了“社区共识”。本周不行动，下周别人就会推出更好的方案。

**$19 报告值不值？** 如果它能帮你节省 2 小时/周，那它值 $40/月。如果它能让你不用重复解释项目结构，那它值 $49/月。所以，一个 $9/月的“AI 代理记忆服务”定价是合理的。

**真正麻烦的活是什么？** 不是写代码，而是定义“记忆”的边界。代理应该记住什么？文件结构、函数签名、上次的错误信息？还是整个对话历史？这个产品设计决策，比实现技术本身更难。

### 🎯 今日 2 小时构建：`AgentRecall` (代理回忆录)

**产品名：** AgentRecall (代理回忆录)

**一句话描述：** 给你的 Claude Code / Cursor 一个本地记忆。它自动记录你的项目上下文、代码决策和错误修复历史，让 AI 代理在下一次对话时“记得”你是谁。

**支撑证据：**
- `Show HN: Recall – Local project memory for Claude Code` 在 HN 获得 127 赞和 81 条评论。
- `Show HN: PMB – local-first memory for AI coding agents over MCP` 获得 28 分。
- 两个项目在同一天出现，且都聚焦于“本地优先”和“MCP 协议”，说明这是一个正在快速形成的共识。

**为什么不选另外两个方向：**

1.  **不选 `html-video` (AI 视频生成)：** 这个领域巨头林立（OpenAI, Runway, Pika），且需要强大的 GPU 和训练数据。作为一个独立开发者，你在视频生成赛道与它们竞争，胜算极低。而“代理记忆”是一个全新的、尚未被巨头占据的细分市场。
2.  **不选 `Oak` (Git 替代品)：** 挑战 Git 这个开发者的基础设施，意味着你需要说服开发者放弃他们使用了 20 年的工具。这需要巨大的信任和社区支持，不是一个周末就能验证的。而“代理记忆”是现有工具的“插件”，准入门槛低得多。

**定价：**
- **免费层：** 100 次记忆/月（让用户上瘾）
- **Pro 层：** $9/月，无限记忆 + 搜索功能。
- **团队层：** $29/月，共享项目记忆 + 团队上下文。

**最快验证路径：**

1.  **今天做的事情：** 创建一个 Google Form，标题是“你的 AI 编码代理有健忘症吗？”。在 Form 里问 3 个问题：
    - 你用的是哪个 AI 编码工具？（Claude Code / Cursor / Copilot / 其他）
    - 你每周因为代理“失忆”浪费多少时间？（&lt;1 小时 / 1-3 小时 / >3 小时）
    - 如果有一个工具能让你的代理“记住”项目上下文，你愿意付多少钱？（$0 / $9/月 / $19/月）
2.  **投放：** 在 HN 上 `Recall` 和 `PMB` 的讨论帖里，找到那些抱怨“每次都要重新解释”的用户，礼貌地私信他们或回复，邀请他们填写 Form。同时，在 Twitter 上搜索“Claude Code memory”或“Agent context”，与这些用户互动。
3.  **判断标准：** 如果 24 小时内收到超过 30 份有效回复，且超过 50% 的人愿意付费，就值得做。如果不到 10 份，放弃。

**MVP 保持人工：** 用 Google Sheet 管理你的用户反馈。不要一开始就写代码。你的第一版产品可以是一个 **Markdown 文件**，里面有“如何手动为 Claude Code 添加记忆”的指南，然后以 $9 的价格出售。这叫“教他们怎么做，而不是替他们做”。

- **信号：** `Crespo` - 一个用 Tree-sitter AST 蓝图替代原始代码给 LLM 的工具 (26 分)
- **白话解读：** 这个工具试图解决一个根本问题：LLM 不理解代码的“结构”，只理解代码的“文本”。`Crespo` 把代码转换成一种 LLM 更容易理解的“蓝图”(AST)，从而提高 AI 生成代码的准确性。
