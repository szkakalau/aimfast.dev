## 📝 主编说

大家都在聊 AI agent 进 Slack、AI 自动修代码、AI 当视频剪辑师。但今天真正可构建的信号藏在两个地方：**Alibaba 开源了一个代码审查工具，7 小时拿了 7,425 个 star**，以及 **一个叫 Goldfish 的 Mac 工具，按 Option 键就能知道你正在做什么并替你回复**——157 条评论里全是"这太可怕了"和"我立刻想要"。

谁会先付钱？**用了 Claude Code 但被 API 账单吓到的独立开发者**，或者**每天花 2 小时回复 Slack 消息的产品经理**。为什么是这周？因为 Anthropic 刚发布了 Fable 模型，26 人在 HN 上抱怨"太贵了"——意味着 Claude Code 用户正在寻找替代方案或省钱方法。

$19 的一次性报告值不值？如果它能帮你每月省下 $50 的 API 费用，答案是肯定的。但真正麻烦的活是：**让用户相信你的工具不会偷走他们的代码**——今天所有高讨论度产品都在打"信任"这张牌。

## 🎯 今日 2 小时构建

### CodeReview Lite — 给你团队的 AI 代码审查员

**一句话**：一个轻量级的代码审查工具，在你 push 到 GitHub 后自动跑规则检查（空指针、线程安全、SQL 注入），并且给出代码行级别的修改建议。

**支撑证据**：Alibaba 的 open-code-review 在 GitHub 上 7 小时获得 7,425 star，449 fork。它用混合架构（确定性规则 + LLM Agent），内置了阿里巴巴内部验证过的规则集。用户正在以惊人的速度拥抱这个方向。

**为什么不选另外两个候选方向？**

1. **Goldfish（Mac AI 助手）**：157 条评论说明需求真实，但这是一个客户端产品，需要 macOS 开发经验，2 小时做不出来。而且隐私争议大——"它知道你所有工作"这件事本身就吓跑了一半用户。
2. **MakersClaw（Slack AI 员工）**：320 票看着不错，但这是 SaaS 产品，需要后端 + Slack API 集成 + 支付系统。2 小时连 MVP 的边都摸不到。

**CodeReview Lite 的优势**：你可以用 GitHub Actions + 一个 Markdown 文件 + 调用 Claude API 在 2 小时内跑通。不需要服务器，不需要数据库。

**定价**：
- **$19 一次性报告**：生成你代码库的"健康检查报告"，指出 10 个最严重的代码问题
- **$9-29/月监控**：每次 push 自动审查，按仓库数量分级

**最快验证路径（今天就能做）**：
1. 创建一个 GitHub Action 模板仓库，取名 `code-review-lite`
2. 写一个 100 行的 YAML 文件，调用 Claude API 审查 PR
3. 在 HN 的 "Show HN: 我花 2 小时做了一个代码审查工具" 发帖
4. 看有没有人 star 和 fork

**MVP 保持人工**：第一版就是一个 Google Form 收集代码仓库地址，你手动跑审查，然后发 Markdown 报告。不需要自动化。

**Counter-view**：Alibaba 的工具是开源的，如果你只做 wrapper，用户可以直接用原版。你需要加差异化——比如更简单的配置、更便宜的定价、或者专注于特定语言（如 Go 或 Rust）。

## 🔍 发现机会

### 🆕 Solo-founder 产品发布

**Goldfish: Press Option. It knows your work and replies like you**
- **信号**：Product Hunt 490 票 / 157 评论，34 分
- **白话解读**：一个 Mac 应用，你按 Option 键，它就知道你正在做什么（看邮件、写文档、聊天），然后替你生成回复。用户评论两极分化——有人觉得"这是终极生产力工具"，有人觉得"这是终极隐私噩梦"。
- **关键判断**：这个产品的成功取决于它能不能让用户信任。如果它能做到"所有数据本地处理，不上传云端"，就能消解一半的隐私担忧。
- **Counter-view**：macOS 的辅助功能权限是出了名的难申请。Apple 可能会在审核时卡住。而且，如果用户每 5 分钟就要按一次 Option 键，它就不是"无感"的。

**Invoko: A little hand on your Mac**
- **信号**：Product Hunt 30 分
- **白话解读**：又一个 Mac AI 助手，但定位是"小手"而不是"大脑"——帮你点按钮、填表单、做重复操作。
- **关键判断**：和 Goldfish 的区别是定位更窄——不是"替你思考"，而是"替你动手"。这反而可能更安全（不需要读取你的内容，只需要模拟点击）。
- **Counter-view**：macOS 的自动化权限（AX API）正在收紧。Apple 可能会在未来版本中限制这种工具。

### ⭐ GitHub 快速增长开源项目

**alibaba/open-code-review**（7,425 star / 7 小时）
- **信号**：阿里巴巴开源的代码审查工具，混合架构（确定性规则 + LLM Agent），内置 NPE（空指针异常）、线程安全、XSS（跨站脚本攻击）、SQL 注入规则集
- **白话解读**：这不是又一个"AI 帮你写代码"的工具。它是"AI 帮你找代码里的 bug"——而且规则是阿里巴巴内部用了多年的，不是 AI 瞎猜的。
- **关键判断**：代码审查是少数几个 AI 可以明确说"我找到了这个 bug"的场景。不像"帮我优化代码"那样模糊。这意味着它可以按"找到的 bug 数量"来定价。
- **Counter-view**：开源意味着任何人都可以自建。你的收费版本必须提供"开箱即用的规则集"或"更简单的配置"才能卖出去。

**garrytan/gstack**（30 分）
- **信号**：Garry Tan（YC CEO）开源了他的 Claude Code 配置——23 个定制工具，让 AI 代理扮演 CEO、设计师、工程师等角色
- **白话解读**：YC 的掌门人公开了他的"AI 工作流"。这不是一个产品，而是一个教程式的配置仓库。但它信号强烈——连 Garry Tan 都在用 Claude Code 做全栈开发。
- **关键判断**：这个仓库的价值不是代码，而是"方法论"。你可以把它包装成"YC CEO 同款 AI 工作流"的付费课程或模板。
- **Counter-view**：Garry Tan 的配置是为他自己定制的。普通人直接套用可能水土不服。而且，如果 Claude Code 本身更新了 API，这些配置可能立刻失效。

**MemPalace/mempalace**（55,747 star）
- **信号**：号称"最好的基准测试开源 AI 记忆系统"，免费
- **白话解读**：让 AI 记住你和它的对话历史，而不是每次重新开始。这是所有"AI 助手"类产品的基础设施。
- **关键判断**：55k star 说明这个需求巨大。但问题是——开源记忆系统意味着任何人都可以自建。真正的商业机会在"托管记忆服务"——你不必自己部署，付 $9/月就用。
- **Counter-view**：记忆系统是 AI 基础设施，不是面向终端用户的产品。如果你不做开发者工具，这个信号对你用处不大。

### 📈 增长最快的开发者工具

**cc-switch: Cross-platform All-in-One assistant for Claude Code, Codex, OpenCode**（26 分）
- **信号**：一个跨平台桌面工具，让你在 Claude Code、Codex、OpenCode 之间切换

- **Counter-view**：MCP 协议还在早期，API 不稳定。而且，Chrome DevTools 的受众是前端开发者，市场相对狭窄。

### 🔓 开源 AI 重要进展

**github/spec-kit**（28 分，112,676 star）
- **信号**：GitHub 官方出品的"规范驱动开发"工具包
- **白话解读**：GitHub 在推一种新的开发方法论——先写规范（spec），再用 AI 根据规范生成代码。这个工具包帮你入门"spec-driven development"。
- **关键判断**：112k star 说明这个方向有巨大关注。但"规范驱动开发"本身不是一个产品，而是一个方法论。你可以做"AI 规范生成器"——输入需求，输出规范，然后让 AI 根据规范写代码。
- **Counter-view**：大多数开发者讨厌写文档，更讨厌写规范。除非 AI 能自动从对话中生成规范，否则这个方法论可能只停留在理论层面。

**obra/superpowers**（28 分，229,969 star）
- **信号**：一个"agentic skills framework"（AI 代理技能框架）——让 AI 代理具备特定技能的开发方法论
- **白话解读**：这相当于"AI 代理的大学课程"——教 AI 代理如何执行特定任务（比如"写一个 React 组件"或"调试一个 Node.js 服务"）。
- **关键判断**：229k star 说明这是目前最热门的 AI 代理开发框架之一。如果你做"AI 代理技能市场"——开发者上传训练好的 AI 技能，其他人付费下载——就是一个 marketplace 机会。
- **Counter-view**：技能框架和 AI 代理本身一样，技术栈在快速变化。今天用 superpowers，明天可能就用另一个框架。

- **关键判断**：如果你做"GitHub Actions 模板市场"——提供常见的 CI/CD 模板，按使用量收费——就能抓住这个迁移潮。
- **Counter-view**：GitHub Actions 的免费额度对大多数独立开发者已经够用。付费模板的市场可能有限。

**"I Built a Free Open-Source Alternative to Sourcegraph — Here's Why"**（DEV，26 分）
- **信号**：一个开发者做了 Sourcegraph（代码搜索工具）的开源替代品
- **白话解读**：Sourcegraph 是一个付费的代码搜索工具，价格不菲。有人做了开源替代品，说明开发者对代码搜索工具的定价不满。
- **关键判断**：代码搜索是一个明确的痛点——大型代码库中找代码非常痛苦。如果你做"轻量级代码搜索"工具，定价 $19/月（Sourcegraph 的零头），就能抓住小型团队。
- **Counter-view**：Sourcegraph 的免费版已经可以满足小团队需求。而且，VS Code 的内置搜索功能也在不断改进。

## 📈 趋势判断

### 🔥 本周最常见技术关键词及变化

**"Agent" 和 "Code Review" 是今天的高频词**——出现在至少 5 个高评分信号中。从上周的"AI coding agent"（AI 编码代理）变成了"AI code review"（AI 代码审查），说明市场正在从"AI 写代码"转向"AI 审代码"。

**变化方向**：从"创造"到"验证"——用户不再满足于 AI 生成代码，他们想要 AI 保证代码质量。这是一个更实际的场景，因为"审代码"比"写代码"更容易衡量质量。

**Garry Tan（YC CEO）开源了他的 Claude Code 配置**——这说明 YC 在认真对待"AI 代理开发"这个方向。Garry Tan 本人用 Claude Code 做全栈开发，意味着 YC 投资的初创公司也会被鼓励使用类似工具。

### ❄️ 降温的 AI 搜索词

**"AI evaluation" 搜索量下降 77%（当前值 15）**

这个关键词在 3 个月前达到峰值（100），现在跌到了 15。说明"AI 评估"这个概念的炒作已经过去。如果你在做"AI 模型评估"相关的产品，可能需要重新考虑方向。

**为什么降温**：可能是因为市场上出现了太多"AI 评估"工具，用户已经厌倦了。或者，用户发现"AI 评估"本身就是一个伪需求——你不需要评估 AI，你只需要用 AI。

### 🆕 新词雷达

**"Spec-Driven Development"（规范驱动开发）**——GitHub 的 spec-kit 在推这个概念。从零升起，目前只有 28 分，但 112k star 说明关注度很高。

**白话解释**：先写规范（spec），再让 AI 根据规范生成代码。这和传统的 TDD（测试驱动开发）类似，但用 AI 替代了手动编码。

**关键判断**：如果这个概念火了，做"AI 规范生成器"就是一个明确的机会——输入需求，输出规范，然后让 AI 根据规范写代码。

**Counter-view**：大多数开发者讨厌写文档。除非 AI 能自动从对话中生成规范，否则这个概念可能只停留在理论层面。

### 💰 定价和变现模型研究

**今日最佳定价参考**：
- 一次性报告：$19（"不用想"的价格点）
- 月订阅：$9-29（独立开发者能接受的范围）
- 年订阅：$99-299（给用户"省钱"的感觉）

**变现模型选择**：
1. **按使用量收费**：适合代码审查（找到的 bug 数量）
2. **按用户数收费**：适合团队工具（5 人以下免费，5 人以上收费）
3. **按功能收费**：基础功能免费，高级规则集收费

**关键发现**：今天所有高评分产品都是"免费试用 + 付费升级"模式。Goldfish 是"Early Access"（早期访问），Novu 是"Open Source + Cloud"（开源 + 云服务）。说明市场正在从"一次性付费"转向"订阅制"。

---

### 🔄 今天最反直觉的发现

**最反直觉的发现：代码审查比代码生成更热。**

直觉上，"AI 写代码"应该是更大的市场。但今天的信号显示，"AI 审代码"的讨论量和参与度更高。Alibaba 的 open-code-review 在 7 小时内获得 7,425 star，而大多数"AI 编码代理"工具只有几百个 star。

**为什么**：因为"审代码"比"写代码"更容易衡量价值。你可以说"这个工具找到了 5 个空指针异常"，但很难说"这个工具写的代码质量提高了 20%"。可衡量 = 可定价 = 可销售。

**启示**：做"可衡量的 AI 工具"比做"模糊的 AI 工具"更容易变现。如果你在 AI 产品上找不到明确的计量单位，可能方向不对。

### 🎮 Product Hunt 与开发者工具重叠点

**今日 Product Hunt 前 5 名全是开发者工具**——Goldfish（Mac 助手）、Novu Connect（Agent 部署）、PandaProbe（Agent 工程）、MakersClaw（AI 员工）、AutoEdit（AI 视频编辑）。

**重叠点**：这些工具都在解决同一个问题——"如何让 AI 在用户已有的工作流中发挥作用"。不是让用户学新工具，而是让 AI 出现在用户已经打开的应用里（Slack、Teams、Telegram、Premiere Pro、Mac 菜单栏）。

**行动建议**：如果你要做开发者工具，不要做一个"全新的 AI 平台"。做一个"AI 插件"——嵌入到用户已经在用的工具里。
