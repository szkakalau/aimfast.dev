## 🎯 Today's 2-Hour Build: SpecSheet — A Spec Sheet Template for AI Code Output

**Product Name:** SpecSheet

**One-Liner:** A reusable Markdown template you paste into Cursor/Codex's prompt, making AI-generated code come with comments, type definitions, and error handling — no more "it runs but is unreadable" garbage.

**Supporting Evidence:**
- Fission-AI/OpenSpec hit **56,287 stars** (3,934 forks) on GitHub, introducing the concept of "Spec-driven development"
- Ask HN "What has been bothering you lately?" has **152 comments**, with multiple complaints about "AI-generated code has no comments" and "refactoring it feels like archaeology"
- w2solo article "AI-generated code just needs to run? These 5 traps will blow up eventually" sparked **20 points** of discussion, centered on "runs vs. maintainable"

**Why Not the Other Two Directions?**
1. ❌ **Oak (Git alternative)**: Requires fighting the Git ecosystem, at least 6 months of development, and massive user migration costs. You can't do this alone.
2. ❌ **TikZ Editor**: Targets LaTeX users (academia), a tiny market with existing competitors like Overleaf. You can't outcompete them.

**Pricing Model:**
- **$19**: One-time purchase, includes 5 scenario templates (Web app, API, CLI tool, data processing, UI component)
- **$9/month**: Subscription, weekly template updates, community review access
- V1: You manually write 5 Markdown files and sell them on Gumroad

**Fastest Validation Path (Doable Today):**
1. Write a 500-word post titled: "I used a spec sheet to make Cursor-generated code 80% more maintainable"
2. Post on Hacker News (Ask HN), Reddit (r/cursor, r/ClaudeAI), and V2EX programmer boards
3. Include a Google Form in the post to collect emails, promising "I'll send the first template for free within 3 days"
4. If you collect **50 emails** in 24 hours, spend 2 hours writing the template; otherwise, abandon it

**Keep MVP Manual:** Don't build a website or SaaS. Google Form + Markdown files + Gumroad link, all manual delivery. Validate demand before considering automation.

## 📊 Today's Top 3 Signals

### Signal 1: AI Code Quality Crisis — Developers Shift from "It Runs" to "It's Maintainable"

**Composite Observation:** Fission-AI/OpenSpec (56,287 stars) + Ask HN complaint thread (152 comments) + w2solo article (20 points) + "AI code quality" search trend (Google Trends up 120% over the past 3 months)

**Cross-Reference Sources:**
- GitHub Trending: Fission-AI/OpenSpec (56,287 stars)
- Hacker News: Ask HN "What has been bothering you lately?" (152 comments)
- w2solo: "AI-generated code just needs to run? These 5 traps will blow up eventually"
- Reddit: Multiple r/programming posts complaining about AI code maintainability

**Plain English:** Over the past year, everyone thought it was fine if AI could just write code. Now, those who've used AI in real projects for 3-6 months are discovering — **AI-generated code is like disposable chopsticks; you use it once and throw it away, but real projects need repeated modifications**. OpenSpec's explosion shows "spec-driven development" (write a spec first, then let AI write code) is becoming the new paradigm.

**Key Judgment:** This is a **tool + education** dual opportunity. On the tool side, indie developers can build an "AI code quality checker" (like ESLint but for AI output). On the education side, an "AI Code Best Practices Guide" or "Spec Template" has paid value.

**Reverse Perspective:** If AI models themselves dramatically improve code quality within 3-6 months (e.g., GPT-5 natively supports commented output), this demand could disappear. But given current model capability ceilings, there's at least a 12-month window.

---

### Signal 2: Agent Infrastructure Is Being Redefined — From Git to MCP to Local Debugging

**Composite Observation:** Oak (Git alternative, 185 comments) + PMB (local-first MCP memory system, 28 points) + RLM-based debugger (AI agent trace debugging, 26 points)

**Cross-Reference Sources:**
- Hacker News: Show HN Oak (185 comments)
- Hacker News: Show HN PMB (28 points)
- Hacker News: Show HN RLM-based debugger (26 points)
- V2EX: Multi-agent collaboration framework open-source project (28 points)

**Plain English:** Today, 4 different projects are trying to solve the same problem — **the infrastructure for AI agents (software that can call tools on behalf of users) is terrible**. Git isn't suited for agent version management, there's no good local memory system, and debugging is limited to logs. It's like mobile development in 2010 — everyone's building wheels, but there's no standard answer yet.

**Key Judgment:** This direction is **too early** for indie developers. Infrastructure wars are a game for giants and open-source foundations. But you can build a "manual for agent toolchains" — like "How to choose the best memory system for your agent" or "Agent debugging best practices."

**Reverse Perspective:** If Anthropic or OpenAI directly build these features (e.g., Claude with built-in version management), all third-party projects will die. This risk is extremely high in the agent infrastructure space.

### Signal 3: Token Cost Anxiety — Developers Start Counting Every Penny

**Composite Observation:** V2EX post "Are tokens now as expensive as rent?" (9 replies) + w2solo article "Codex is getting more expensive, I built a complete mini-program with free 100 million tokens" (26 points) + headroomlabs-ai/headroom (compresses tool output to reduce token consumption, 26 points)

**Cross-Reference Sources:**
- V2EX Programmer: "Are tokens now as expensive as rent?"
- w2solo: "Codex is getting more expensive"
- GitHub Trending: headroomlabs-ai/headroom (compresses tool output)
- Reddit: Multiple r/LocalLLaMA posts discussing "how to reduce token consumption"

**Plain English:** When AI usage shifts from "playing around" to "daily tool," developers realize — **every API call is burning money**. The popularity of headroom (a project that compresses tool output to reduce tokens) shows developers are willing to spend time optimizing token usage because the savings are real.

**Key Judgment:** The opportunity lies in **token cost visualization tools** — like a browser extension that tells you "this Cursor session cost $X" or "this prompt can be optimized to save 30% tokens." Priced at $9/month for a monitoring tool, targeting heavy AI users spending $100+/month.

**Reverse Perspective:** Model prices keep dropping (GPT-4o is 10x cheaper than GPT-4). If price drops outpace usage growth, this demand could naturally disappear. But for at least the next 6 months, price drops won't outpace usage growth.
