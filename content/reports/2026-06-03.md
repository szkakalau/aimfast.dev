# 📝 主编说

大家都在聊 AI agent 多智能体协作和 Chrome DevTools 接入编程助手——这是今天最热的两个话题。但真正可构建的信号藏在两个地方：一是 **Eyeball**（HN 75 条讨论）——一个能让 AI 直接操控视觉元素的工具，它暗示了“AI 操作 UI”这个方向正在从实验走向实用；二是 **microsoft/SkillOpt**（GitHub 4,486 star/25 天）——微软开源了一个让 LLM 学会“技能”而不只是“提示词”的框架，这意味着**可复用的 AI 技能市场**可能比 agent 框架本身更早变现。

谁会先付钱？**用 AI 做自动化测试的 QA 工程师**和**管理多个 AI agent 的独立开发者**。为什么是这周？因为 SkillOpt 和 Chrome DevTools MCP 同时出现在 GitHub Trending，意味着“AI 技能可复用”这个基础设施正在到位。$19 一份的“AI 技能审计报告”或“Chrome 操作技能包”能卖出去。真正麻烦的活不是写代码，而是**定义哪些 UI 操作可以被标准化成技能**——这需要你打开浏览器，记录自己重复做的 10 个操作，然后封装成可复用的“技能”。

# 📊 今日 Top 3 信号

## 信号 1: AI 操作浏览器 → 技能可复用化

**复合观察**: Chrome DevTools MCP（42,644 star）+ Eyeball（HN 75 讨论）+ microsoft/SkillOpt（4,486 star）→ 三条线指向同一个方向：**AI 操作浏览器的能力正在从“一次性实验”变成“可复用的技能”**

- Chrome DevTools MCP 让 AI 能直接调用 Chrome DevTools 的 API
- Eyeball 让 AI 能“看”到 UI 元素并操控
- SkillOpt 让这些操作可以被训练、验证、部署

**白话含义**: 以前你让 AI 帮你填个表单，它得从头学；现在有了这些工具，你可以把“填表单”这个技能保存下来，下次直接复用。

**为什么不是“又一个 agent 框架”**: 因为 agent 框架解决的是“如何让多个 AI 协作”，而技能复用解决的是“如何让一个 AI 做同一件事更快”。后者更接近付费场景——企业愿意为“减少重复劳动”付钱，而不是为“花哨的多 agent 演示”付钱。

## 信号 2: 独立开发者正在从“做工具”转向“做技能”

**复合观察**: w2solo 上 Lando（Landing Page 生成器）和 Omni AI Pro（AI 工具导航）都是“帮别人省时间”的工具，但 Eyeball 和 Textile 是“帮 AI 省时间”的工具 → 趋势在变化

- Lando: 帮你快速生成 Landing Page（28 分，但单平台）
- Textile: 帮你整理文本片段（28 分，但单平台）
- Eyeball: 帮 AI 看懂 UI（30 分，HN 75 讨论）

**白话含义**: 独立开发者正在从“为人做工具”转向“为 AI 做工具”。后者是新兴市场，竞争更少。

**关键判断**: 如果你现在开始做“AI 技能”相关的工具（技能包、技能审计、技能市场），你有 3-6 个月的窗口期。

## 信号 3: 开发者工具“大公司化”趋势加速

**复合观察**: Microsoft（SkillOpt）+ Google（Chrome DevTools MCP）+ GitHub（spec-kit 107k star）→ 大公司正在系统性地开源开发者工具

- SkillOpt: 微软开源，让 LLM 学会技能
- Chrome DevTools MCP: Google 官方，让 AI 操作浏览器
- spec-kit: GitHub 官方，规范驱动开发

**白话含义**: 大公司正在为 AI 开发者铺基础设施。对独立开发者来说，这不是威胁——**大公司铺路，你在路上开店**。

**关键判断**: 不要和大公司抢基础设施（不要做“又一个 MCP 服务器”或“又一个 agent 框架”），要做**基础设施之上的垂直应用**（比如“Chrome 操作技能包”、“AI 技能审计报告”、“浏览器操作模板市场”）。

# 🔍 发现机会

## Solo-founder 产品发布

### Eyeball – 让 AI 直接“看” UI 的工具

**🔍 信号**: Show HN: Eyeball（236 赞 / 75 评论）——一个让 AI 能直接操控视觉元素的工具。不是截图分析，而是真的让 AI 看到 UI 元素的坐标、类型、状态。

**白话解读**: 以前 AI 操作浏览器靠的是“猜”——它不知道按钮在哪里、输入框有多大。Eyeball 让 AI 能“看”到 UI 的结构，就像人眼看到网页一样。这意味着**自动化测试、数据抓取、表单填写**这些场景可以做得更准、更快。

**关键判断**: Eyeball 本身是一个工具，但真正的产品机会在**使用 Eyeball 的技能包**——比如“用 Eyeball 自动登录 10 个网站并截图”的技能包。$19 一份，卖给做竞品分析的独立开发者。

**反向视角**: Eyeball 是个人项目，可能不维护。如果它挂了，依赖它的技能包也挂了。解决方案：技能包写成通用格式，不绑定特定工具。

### Textile – 文本片段编织工具

**🔍 信号**: Show HN: Textile（36 赞 / 21 评论）——一个桌面应用，帮你把分散的文本片段“编织”到一起。

**白话解读**: 你写文章、写代码、写文档时，经常需要从多个地方复制粘贴文本片段。Textile 让你像拼图一样把片段组合起来。这是 Notion、Obsidian 的细分市场——**专注于“文本片段管理”**。

**关键判断**: 21 条评论说明需求存在但不大。不建议做类似产品，但可以观察它的用户反馈——如果用户抱怨“不能导出技能包”，那就是你的机会。

**反向视角**: Notion 和 Obsidian 都有类似功能，Textile 的差异化可能不足以留住用户。

### CycBox – 物联网调试工具

**🔍 信号**: V2EX 产品发布（0 回复）——跨平台物联网调试工具，支持串口、TCP、UDP、MQTT、Modbus、Lua 脚本、高精度时钟。

**反向视角**: 0 回复也可能是发布时机不对（比如周末发布）。但证据不足，放弃。

**关键判断**: 这是今天最重要的基础设施信号。如果你要做 AI 自动化测试或 AI 浏览器操作，必须基于这个工具。它比 Puppeteer、Playwright 更底层、更灵活。

**白话解读**: 半年前，大家都在讨论“AI 审计”——如何确保 AI 系统公平、透明、安全。但现在热度大幅下降。可能是话题过时了，也可能是 AI 审计变成了企业内部的常规工作。

**关键判断**: 如果你在做 AI 审计工具，现在是时候转型了。但注意：**“AI 技能审计”是不同方向**——不是审计 AI 本身，而是审计 AI 的技能质量（比如“这个 agent 的技能准确率如何”）。

**反向视角**: Google Trends 数据可能不准确，或者“AI audit”被其他关键词替代了（比如“AI governance”）。

## 新词雷达

### “Skill” – 从零升起

**🔍 信号**: SkillOpt（4,486 star）+ OpenClaw Skills（5,400+ 技能）+ 本周多个技能相关项目同时出现。

**白话解读**: “Skill”正在成为 AI 领域的新关键词。以前我们说“prompt”（提示词），现在说“skill”（技能）。区别在于：prompt 是一次性的，skill 是可复用的。

**关键判断**: 这是今年最重要的新词之一。**“AI 技能”可能像“App Store”一样，成为一个新生态**。独立开发者可以在这个生态里做技能包、技能市场、技能审计工具。

**反向视角**: “Skill”可能只是“prompt engineering”的重新包装。如果大公司推出官方技能市场，独立开发者可能被挤出。

### 本周六：记录 10 个浏览器操作，封装成技能包

   - 登录 Notion → 创建新页面 → 填入模板
   - 登录 LinkedIn → 查看消息 → 回复
   - 登录 GitHub → 查看 Issues → 关闭已解决

2. **用 Chrome DevTools MCP 把每个操作写成 JSON 脚本**（45 分钟）
   - 参考官方文档：https://github.com/ChromeDevTools/chrome-devtools-mcp
   - 每个操作写成一个独立的 JSON 文件

   - 定价：$19/个，$29/3 个

## 定价和变现模型研究

### 技能包的定价锚点

|------|------|------|---------|
| 基础包 | $19 | 10 个常用浏览器操作技能 | 独立开发者 |

**为什么是这个定价**:
- $19 是开发者愿意“试错”的价格（一杯咖啡 + 一顿饭）
- $49 是团队采购的“无需审批”上限
- $199/月是企业预算的“低门槛”

**验证路径**:
1. 先卖 $19 的基础包
2. 如果卖出 10 份，升级到 $49 的专业包
3. 如果卖出 50 份，考虑企业包

## 今天最反直觉的发现

### 反直觉：AI agent 框架（107k star）不如 AI 技能包（4k star）值得做

**为什么反直觉**: 107k star 的 agency-agents 看起来更热、更有前景。但 4k star 的 SkillOpt 才是真正的机会。

**为什么是对的**:
1. **竞争密度**: agency-agents 是红海（AutoGPT 150k star、CrewAI 60k star），而 SkillOpt 是蓝海（只有微软在做）
2. **变现路径**: agent 框架是基础设施，很难直接收费；技能包是垂直应用，可以直接卖 $19
3. **维护成本**: agent 框架需要持续维护（232 天还在更新），技能包是“一次写、多次卖”
4. **用户画像**: agent 框架用户是“想玩”的开发者，技能包用户是“想省时间”的开发者——后者更愿意付钱

**结论**: 不要被 star 数迷惑。**4k star 的蓝海 > 107k star 的红海**。

## Product Hunt 与开发者工具重叠点

### 今日重叠点：MCP 上下文 + 开发者文档

**🔍 信号**: Moxie Docs（PH 26 分）+ Paste MCP（PH 26 分）+ Chrome DevTools MCP（GH 42k star）

**白话解读**: MCP 上下文正在成为开发者工具的新标配——你的工具需要让 AI 能理解它。Moxie Docs 让文档变成 AI 可读的，Paste MCP 让剪贴板变成 AI 可读的，Chrome DevTools MCP 让浏览器变成 AI 可读的。

**关键判断**: 如果你在做任何开发者工具，**现在就需要支持 MCP 协议**——否则你的工具会被 AI 忽略。这是一个“不做就淘汰”的技术标准。

**反向视角**: MCP 是 Anthropic 推出的协议，Google 和微软也在跟进。但它可能被其他协议取代（比如 OpenAI 的 Function Calling）。建议同时支持多个协议。
