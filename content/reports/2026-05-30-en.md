### Agent Permission Auditor

One-liner: A CLI tool that scans your local Agent config files (e.g., Claude Code, Zot) and generates a "permission risk report" telling you which Agents have excessive permissions.

**Supporting Evidence**:
- The `Continue? Y/N` game sparked 152 discussions on HN, proving "permission fatigue" is a real and widespread pain point.

**Why Not the Other Two**:
- **Skip Hallucinate (MMO Rave)**: Highest score, but it's an entertainment product with a single monetization path (ads/sponsorships) and doesn't match a Builder's core skill (engineering).
- **Skip Osiris (OSINT Platform)**: It's a full product with frontend, backend, and data pipelines. You can't even set up the environment in 2 hours — not suitable for "quick validation."

**Pricing**:
- **$19 one-time report**: Users upload their config files, you return a PDF report.
- **$9-29/month monitoring**: Continuous Agent permission change monitoring with webhook alerts.

**Fastest Validation Path**:
1.  Create a GitHub repo called `agent-permission-auditor`.
2.  In the README, clearly state: what it does (scans Agent config files), why it's needed (permission fatigue), and what formats it supports (JSON, YAML, TOML).
3.  On the HN `Show HN: Continue? Y/N` discussion thread, reply with a comment: "Fun game. If you're actually worried about Agent permissions, this tool can audit them: `[your GitHub link]`."
4.  If you get 10+ stars or issues, it's worth spending a weekend on an MVP.

**Keep MVP Manual**: Start with a Python script that has a hardcoded rule checker (e.g., check if the `tools` array includes `read_files` permission). Then use a Google Form to collect user config files and manually send back reports. Zero backend cost.
