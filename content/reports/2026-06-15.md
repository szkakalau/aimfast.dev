# AimFast.Dev日报 — 2026-06-15

## 📝 主编说

大家今天都在刷两件事：**Garry Tan 的 gstack**（12.6 万 star）和**阿里开源的代码审查工具**（7000+ star）。表面看是两个 AI 编码工具，但真正可构建的信号藏在更底层——**AI 编码的 Agent Skills 生态正在爆发**。mattpocock 的 `skills` 项目 12.8 万 star、addyosmani 的 `agent-skills`、还有 `claude-code-best-practice`，都在教一件事：**怎么给 AI 写"技能说明书"**。

谁会最先付钱？**用了 Claude Code/Codex 但觉得"AI 不够聪明"的独立开发者**。为什么是这周？因为 gstack 把"CEO 技能""设计师技能"打包成 23 个工具，而社区正在疯狂复制这个模式——但没人做**技能包市场**。一个 $19 的"Claude Code 最佳技能包合集"报告，或者 $9/月的技能包订阅，今天就能开始卖。

真正麻烦的活：不是写代码，是**整理和测试这些技能文件**。但第一版只需要一个 Google Form 收集需求 + 手动整理 Markdown 文件。

## 🎯 今日 2 小时构建 — AgentSkill.store

**产品名**: AgentSkill.store（AI 代理技能商店）

**一句话描述**: 一个专注于 AI 编码 Agent（Claude Code、Codex、OpenCode 等）技能包（Skills）的发现和交易市场。开发者可以购买、出售、评分和分享让 AI 更聪明的技能文件。

**支撑证据**:
- `mattpocock/skills` — 128,906 star（28 分）
- `addyosmani/agent-skills` — 高参与度（28 分）
- `shanraisshan/claude-code-best-practice` — 高参与度（28 分）
- `garrytan/gstack` — 110,160 star，23 个"角色技能"（30 分）
- `obra/superpowers` — 227,934 star，技能框架方法论（28 分）

**为什么不选另外两个方向**:

1. ❌ **不做另一个 AI 代码审查工具**（alibaba/open-code-review）— 阿里已经开源了，企业级功能齐全，个人开发者拼不过。而且需要整合到 CI/CD 流程，验证周期太长。

2. ❌ **不做 AI 记忆系统**（MemPalace）— 已经有 5.5 万 star 的开源方案，市场拥挤。而且"AI 记忆"这个需求太模糊，买方不明确。

**定价**:
- **$19** — "Top 50 Claude Code Skills Pack"一次性报告（含 50 个经过测试的技能文件）
- **$9/月** — 技能包订阅（每周更新 5-10 个新技能 + 社区评分）
- **$29/月** — Pro 版（含自定义技能咨询 + 优先支持）

**最快验证路径**（今天就能做）:

1. 花 2 小时从 `mattpocock/skills`、`addyosmani/agent-skills`、`gstack` 中精选 10 个最实用的技能文件
2. 创建一个 Google Form："你想要什么 AI 编码技能？"（列出 10 个选项 + 自定义输入）
3. 在 HN 上 `mattpocock/skills` 的讨论线程（12.8 万 star 意味着有大量活跃用户）发帖："I curated the best 10 Claude Code skills from top repos — free download"
4. 收集邮箱，如果 48 小时内 100+ 注册 → 做完整产品

**MVP 保持人工**: Google Form 收集需求 → 手动整理 Markdown 文件 → Gumroad 卖 $19 报告。不要一上来就做平台。

### GitHub 快速增长开源项目（无商业版本）

#### 🔍 alibaba/open-code-review — 代码审查工具
- **信号**: 7,001 star / 400 fork，GitHub Trending 32 分
- **白话解读**: 阿里开源了一个代码审查工具。不是纯 AI 审查——它是"确定性管道 + AI 代理"的混合架构。能检测空指针异常（NPE）、线程安全问题、XSS、SQL 注入，并直接在代码行上给出评论。
- **关键判断**: 企业级工具，个人开发者用不上。但如果你在做一个**给小型团队用的简化版代码审查 SaaS**，可以参考它的混合架构。
- **反向视角**: 阿里开源意味着大厂已经免费提供了。小团队做付费版本需要差异化——比如更简单的配置、更少的误报、或者针对特定语言（Rust、Go）的优化。

#### 🔍 AprilNEA/OpenLogi — Logitech 软件替代品
- **信号**: 4,845 star / 93 fork，20 天，Rust 编写
- **白话解读**: 一个用 Rust 写的 Logitech 鼠标配置软件替代品。不需要注册账号，不上传数据，本地运行。能重新映射按钮、调整 DPI（鼠标灵敏度）、设置 SmartShift（罗技的滚轮模式切换）。
- **关键判断**: 这是"大厂软件太烂"的经典机会。Logitech Options+ 是 Electron 写的、强制登录、有遥测。OpenLogi 用 Rust 写、本地优先、无遥测。**但这是开源项目，不是商业产品**。
- **反向视角**: 开源项目已经免费了。商业版本需要增加什么？多平台支持、企业部署、或者**鼠标配置文件云同步**（端到端加密）——但这就违背了"无遥测"的承诺。

### 开发者在抱怨什么

#### 🔍 "Claude Code Token 烧太快" — 月费从 $250 砍到 $50
- **信号**: w2solo 帖子，实测 5 招降低 Claude Code 使用成本
- **白话解读**: 一个开发者发现用 Claude Code 解决一个 bug 能花掉 $40。他分享了 5 个省钱技巧，包括限制上下文长度、使用本地模型做简单任务、设置 Token 预算上限等。
- **关键判断**: 这是**AI 编码成本焦虑**的直接证据。有 $250/月的用户愿意花时间研究怎么砍到 $50。这意味着**AI 编码成本监控/优化工具**有市场。
- **反向视角**: 省钱技巧是"一次性的"。用户学会了就不需要工具了。但如果你能做成**自动优化**——比如自动判断什么任务用 Claude、什么任务用本地模型——可能更有持续价值。

#### 🔍 "PDF 两个文档同步滚动" — 文档比对工具需求
- **信号**: w2solo 帖子，寻找 PDF 同步滚动比对工具
- **白话解读**: 一个开发者需要对比两个 PDF 文档的差异。新版旧版排版不同，手动切换看不过来。他想要一个能**同步滚动、高亮差异**的工具。
- **关键判断**: 小而具体的工具需求。PDF 比对工具很多（Adobe Acrobat、DiffPDF），但免费/好用的不多。**$9 一次性购买的 PDF 比对工具**可能有机会。
- **反向视角**: 这个需求可能被 AI 解决——"给我两个 PDF，告诉我差异"——比手动滚动更高效。而且免费 PDF 编辑器汇总帖子已经列出了多个开源方案。

- **关键判断**: 227k star 说明方法论层面的需求巨大。但"框架"太抽象，无法直接变现。**基于这个框架的实战课程**或**认证**可能是更好的产品形态。

### 开源 AI 重要进展

#### 🔍 cc-switch — Claude Code / Codex / OpenCode 切换器
- **信号**: 26 分，跨平台桌面应用，支持 Claude Code、Codex、OpenCode、Goose、Pythagora 的切换
- **白话解读**: 一个桌面工具，让你在不同 AI 编码 Agent 之间切换。不用退出终端、不用改配置。一键切换底层的 AI 引擎。
- **关键判断**: 这反映了**AI 编码 Agent 碎片化**的问题——用户在不同工具间切换的成本很高。cc-switch 解决了一个真实痛点。但**这是开源项目**，商业模式不清晰。
- **反向视角**: 如果每个 AI 编码 Agent 都开始支持"多引擎"（Claude Code 已经在做），这个工具的价值会迅速归零。

#### 🔍 claude-mem — 跨会话持久上下文
- **信号**: 26 分，捕获 Agent 的完整上下文并在会话间保持
- **白话解读**: 让 AI 编码 Agent 记住之前会话的内容。你昨天让它重构了一个模块，今天继续工作时它还记得上下文和决策理由。
- **关键判断**: 这是"AI 记忆"在编码场景的落地。MemPalace（5.5 万 star）在做通用 AI 记忆，`claude-mem` 在做编码特定记忆。**编码上下文持久化**是一个更具体的切入点。
- **反向视角**: 隐私问题——Agent 的上下文可能包含敏感代码。本地存储 vs 云同步的权衡。而且不同 Agent 的记忆格式不兼容。

### 新词雷达

**Agent Skills（代理技能）** — 本周从零升起的关键词。指**让 AI 编码 Agent 执行特定任务的指令集**，比"提示词"更结构化，比"工具"更可复用。

**白话解释**: 你不再跟 AI 说"帮我写一个排序函数"，而是给它一个"排序函数技能文件"，里面有最佳实践、边界条件、测试用例。AI 加载这个技能后，写排序函数的质量和一致性大幅提升。

**关键判断**: 这个概念正在成为 AI 编码的"新单元"——就像函数是代码的单元、组件是 UI 的单元。**谁定义了"技能"的标准格式和交易市场，谁就掌握了 AI 编码生态的入口**。

### 今天最反直觉的发现

**最反直觉**: **AI 编码的"技能"比"模型"更重要**。

主流叙事是"更好的模型 = 更好的编码"。但今天的数据显示，**社区在疯狂复制和分享的不是模型配置，而是技能文件**。

`mattpocock/skills`（12.8 万 star）是 TypeScript 类型专家写的技能集，不是模型。`gstack`（11 万 star）是 23 个角色技能，不是模型。`obra/superpowers`（22.7 万 star）是技能框架，不是模型。

这意味着：**如果你能帮开发者"让现有的 AI 更聪明"，比"等更好的 AI 出来"更有商业价值**。技能包是"增量价值"——不需要换模型，只需要加一个文件。
