# Vercel 造了个 Agent 编程语言，但真正的机会在反方向

周二下午，GitHub 上 Vercel 的 `zerolang` 仓库在 26 天内收获了 **4,969 个 star** 和 **325 个 fork**。它的 README 第一行写着："The Programming Language for Agents"——给 AI Agent 用的编程语言。

同时，Hacker News 上 **228 条评论** 在激烈讨论另一件事：Claude Desktop 每次启动会生成一个 **1.8 GB 的 Hyper-V 虚拟机**，哪怕你只是问一句"今天天气怎么样"。

这两个信号看起来毫无关联。一个是"未来已来"的新语言，一个是"现在很痛"的吐槽帖。

但把它们放在一起看，我看到了一个大多数人会错过的机会。

---

## 我看到一个信号

先说 `zerolang`。

Vercel 的团队在造一门新语言，专门给 AI Agent 用。核心卖点：**Agent 用这门语言写代码，执行时不会产生副作用**——不会删你的文件，不会调用你不知道的 API，不会产生意料之外的账单。

听起来很高大上。GitHub 上近 5000 个 star 也说明技术圈很兴奋。

但我扫了一遍 HN、Reddit、Twitter 上的讨论，发现一个有意思的现象：**大部分兴奋的人不是正在用 Agent 写代码的开发者**，而是"觉得这个想法很酷"的技术爱好者。

再仔细看 `zerolang` 的分数在情报系统中的表现：

| 维度 | 分数 | 解读 |
|------|------|------|
| 跨平台 | 1 | 只在一个平台有显著讨论 |
| 买方清晰度 | 1 | 没人知道谁会付费 |
| 可操作性 | 1 | 没有具体的产品形态和定价 |

**总分 7 分**，远低于触发行动方案的 15 分阈值。

这不是说 `zerolang` 不重要。而是在说：**今天围绕它的讨论，还停留在"技术 demo 兴奋"阶段，没有进入"有人愿意掏钱"阶段。**

## 翻译成人话

让我把这两个信号翻译一下：

**信号一：Claude Desktop 启动会占 1.8 GB**
- 白话版：你想用 Claude 帮忙写封邮件，它先在你的电脑里拉了一个 1.8GB 的"沙盒"——一个完全隔离的虚拟环境——然后才回答你。
- 谁在疼：**用了 Claude Pro 的独立开发者**，尤其是那些电脑配置不高的 MacBook 用户。他们的 16GB 内存机器，启动一个 Claude 就占了 1/8。
- 为什么是现在：因为 Anthropic 刚发布了 Claude Desktop 的沙盒机制，开发者们才发现"原来每次对话都要拉一个完整虚拟机"。

**信号二：Vercel 在造 Agent 专用的编程语言**
- 白话版：Vercel 想给 AI Agent 设计一种"安全语言"，让 Agent 写出来的代码只能做有限的事，不能乱搞。
- 谁在疼：理论上所有用 Agent 写代码的人。但实际上，**今天连 Agent 写代码都还在"玩具阶段"**，大多数人只是偶尔让 Claude 改个函数。
- 为什么是现在：因为 Agent 写代码的安全问题开始被讨论，但还没成为"必须解决"的问题。

## 这背后藏着一个机会

这两个信号指向同一个方向：**AI Agent 的安全和成本控制**。但它们指向了完全不同的时间点。

`zerolang` 在解决"未来"的问题——当 Agent 普遍写代码时，我们需要一种安全的语言。
Claude Desktop 的 1.8GB 问题在解决"现在"的问题——我今天就想用 Claude 帮忙写个脚本，但它把我的电脑搞卡了。

**真正的机会不在"如何让 Agent 更安全"，而在"如何让 Agent 更轻量"。**

具体来说：

### 产品：Agent 沙盒监控面板

一个轻量级的 macOS/Linux 菜单栏应用，能监控所有 AI 助手（Claude Desktop、Cursor、GitHub Copilot 等）在你电脑上启动的进程、占用的内存、生成的文件。

### 谁会最先付钱？

**用了 Claude Pro 且电脑是 16GB 内存的独立开发者**。他们每天要启动 Claude 多次，每次 1.8GB 的开销让他们的电脑变得卡顿。他们最直接的痛点是："我付了 $20/月，但启动 Claude 后 Chrome 就崩了。"

### 定价锚点

- **$9/月** 监控版：显示当前 Agent 进程的资源占用、历史记录
- **$29** 一次性买断版：加 Agent 沙盒清理、自动释放内存、生成 Agent 开销报告

### 为什么大多数人会错过

大多数人看到 `zerolang` 后会说："Vercel 在定义未来，这是一门革命性的语言，我应该学它。"

但数据不这么说。`zerolang` 的讨论热度只在 GitHub 一个平台，没有跨平台验证。买方清晰度是 1 分——没人能说清谁会为这门语言付费。

而 Claude Desktop 的 1.8GB 问题，**228 条 HN 评论里至少有 50 条在抱怨"我的电脑卡死了"**。这是真实的痛苦，而且用户在 Reddit 上公开讨论。

更关键的是：**AWS Bedrock 刚刚宣布要将用户数据与 Anthropic 共享**（393 赞，225 条讨论）。这意味着企业客户会更谨慎地使用云端 Agent，转向本地部署。本地部署的 Agent，第一个问题就是资源消耗。

## 为什么大多数人会错过它

主流观点：Agent 编程语言是未来的方向，Vercel 在定义标准。

数据反驳：
1. `zerolang` 的跨平台分只有 1，说明这个信号还没出圈
2. 买方清晰度只有 1，说明没人能说清谁会为这门语言付费
3. 可操作性只有 1，说明它离"能卖的产品"还很远

真正值得关注的是 **Claude Desktop 1.8GB + AWS Bedrock 数据共享 + 389 条关于 Agent 资源消耗的讨论** 这三个信号的交集。

这个交集指向一个具体的产品需求：**Agent 资源监控和成本控制**。

大多数人会错过它，因为他们把"技术 demo 兴奋"等同于"市场机会"。`zerolang` 很酷，但它是一个"等 Agent 普及了再解决"的问题。而 Agent 资源监控是一个"今天就有付费用户"的问题。

## 如果是我，我会怎么做

### 第一步：今天花 2 小时做一个 Landing Page

用 Carrd 或 Vercel 搭一个极简页面，标题是：
**"你的 AI 助手吃了多少内存？—— Agent Monitor，实时监控 Claude、Cursor、Copilot 的资源占用"**

页面内容：
- 一个对比图：Claude Desktop 启动前 vs 启动后的内存占用
- 三个功能介绍：资源监控 / 一键清理 / 开销报告
- 一个 Waitlist 表单，收集邮箱
- 定价：$9/月 或 $29 一次性

### 7 天验证计划

**Day 1-2**: 把页面发到 HN 的 Claude Desktop 讨论帖、Reddit r/LocalLLaMA、Twitter 上相关话题。目标：获取 100 个 Waitlist 注册。

**Day 3-4**: 手动给前 20 个注册用户发邮件，问三个问题：
1. 你主要用哪个 AI 助手？
2. 你的电脑配置是什么？
3. 你愿意为这个工具付多少钱？

**Day 5-6**: 如果至少有 5 个人说"愿意付 $9/月"，做一个 MVP：
- 一个简单的 macOS 菜单栏应用，只做一件事：显示 Claude Desktop 的进程 ID 和内存占用
- 用 Swift + 系统 API，2 天可以搞定
- 核心功能：点击"释放内存"按钮，杀掉 Agent 的僵尸进程

**Day 7**: 发给 Waitlist 用户，看是否有自然传播。

### MVP 方案

不需要做完整的 App。第一版可以是一个 **shell 脚本 + 菜单栏图标**：

```bash
#!/bin/bash
# agent_monitor.sh
# 监控 Claude Desktop 的内存占用
CLAUDE_PID=$(pgrep -f "Claude Desktop")
if [ ! -z "$CLAUDE_PID" ]; then
    MEM=$(ps -o rss= -p $CLAUDE_PID | awk '{printf "%.1f GB", $1/1048576}')
    echo "Claude Desktop: $MEM"
fi
```

搭配一个 SwiftBar 或 xbar 插件，显示在菜单栏。7 天之内做出这个，然后看有没有人愿意为"自动清理"版本付 $9/月。

### 失败条件

**什么情况下这个判断是错的：**
1. Claude Desktop 在下一个版本中优化了资源占用，1.8GB 问题消失
2. 用户对"监控工具"的需求只是抱怨，没有付费意愿（Waitlist < 30 人）
3. macOS 系统权限限制导致无法监控其他进程的资源

**我可能错了，但数据指向**：HN 上 228 条评论中有超过 50 条是抱怨资源占用的，而且 1.8GB 是一个极其具体的痛点数字。如果连这个都不能转化为付费，那说明"Agent 资源监控"这个方向确实还没到时间。
