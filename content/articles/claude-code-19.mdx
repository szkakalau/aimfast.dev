# 被 Claude Code 封禁之后，我看到了一个 $19 的产品机会

周二下午，Hacker News 上一个帖子获得了 78 个赞和 87 条评论。标题很直白："Ask HN: Anthropic banned me from using Claude Code and I don't know what to do"（我被 Anthropic 禁止使用 Claude Code 了，不知道该怎么办）。

发帖人 ayi 说，他正在用 Claude Code 写一个项目，突然收到邮件——账户被暂停了。没有解释，没有申诉通道，没有提前警告。他的开发流程直接中断了。

87 条评论里，有人骂 Anthropic 的 moderation 政策太激进，有人教他怎么写申诉邮件，有人给他推荐替代方案（Cursor、Continue.dev、Aider）。但有一个声音让我停了下来——好几个人的回复可以概括成一句话：

**"所以我一直不用 Claude Code。我就用 API 自己写个脚本，或者用 openai-compatible 的接口，这样谁也管不着我。"**

这不是一个投诉帖。这是一个需求信号。

---

## 翻译成人话

先解释一下发生了什么。

Claude Code 是 Anthropic 推出的一款终端工具，让开发者可以在命令行里直接和 Claude 对话，让它写代码、读文件、执行命令。它本质上是一个"AI 程序员助手"，但绑定的是 Anthropic 的账户和服务条款。

发帖人遇到的问题：Anthropic 单方面封了他的账号。可能的原因包括检测到自动化使用、付费异常、或者内容审查触发。但关键是——**没有透明的规则，也没有申诉渠道**。

这就像你租了一套房子，房东突然说"你明天搬走"，不留解释，不给你时间找新房。

### 谁在疼？

**重度使用 AI 编码工具的独立开发者和小团队。**

这些人每天把 Claude Code、Cursor、Copilot 当作开发环境的一部分。不是偶尔问个问题，而是让 AI 参与整个开发流程——写代码、重构、调试、跑测试。

对于他们来说，账户被封意味着：
- 正在跑的任务直接中断
- 项目中积累的 prompt 历史、配置、偏好全部丢失
- 换工具需要重新配置、学习、适应

### 为什么是现在？

两个原因：

1. **AI 编码工具的普及率已经到了"成瘾"阶段**。不是"用了挺方便"，而是"没有它开发效率减半"。GitHub Copilot 已有 180 万付费用户，Claude Code 和 Cursor 的用户量也在快速增长。依赖越深，封号的代价越大。

2. **平台正在收紧控制**。Anthropic、OpenAI、GitHub 都在加强内容审查和使用限制。不是因为你做了坏事，而是因为他们的 moderation 模型误判了。或者因为你触发了某个模糊的边界。

**定价锚点：$19 一次性。** 这不是一个月度订阅——它解决的问题是"一次性的灾难恢复"，不是持续的监控服务。用户愿意为一次救急付 $19，然后可能再也不需要你。这没关系。

## 这背后藏着一个机会

机会的名字很直白：**"AI 编码工具账户备份与迁移工具"**。

具体来说，是一个工具，能：

1. **导出**你在 Claude Code / Cursor / Copilot 里的所有设置、prompt 历史、项目配置、自定义指令
2. **备份**到本地或你自己的云存储（不是我们的服务器）
3. **导入**到另一个兼容的工具或自建方案中

### 谁会付钱？

**被平台警告过或听说过别人被封的 AI 编码重用户。**

具体角色：
- 独立开发者（接了外包项目，依赖 Claude Code 写代码）
- 小型创业团队（2-5 人，共享一个团队账户）
- 技术博主 / 教程作者（积累了大量的 prompt 和 workflow）

### 为什么大多数人会错过这个？

因为主流叙事在讨论两个方向：
- "Anthropic 的政策太烂了，我们应该抗议"
- "换个工具不就行了，Cursor 也很好用"

这两个方向都没错，但它们忽略了一个中间状态：**用户不是不想换，而是换的成本太高**。他们积累的 prompt、配置、workflow 是有价值的。如果迁移成本降到零，他们随时可以走。

大多数人看到的是"平台封号"这个事件。我看到的是**"迁移成本"**这个产品机会。

### 数据支撑

- HN 上 87 条评论中，至少有 12 条直接表达了"我需要一个替代方案"或"我不敢用 Claude Code 就是因为怕被封"
- 同期 GitHub 上 `mattpocock/skills` 获得了 145k star，这是一个教你怎么写 `.claude` 目录配置的项目——说明用户正在主动学习和定制自己的工作流
- `datawhalechina/hello-agents` 也有 61k star，说明"从零构建 agent"的需求在快速增长

**这三组数据指向同一个方向：用户正在把 AI 工具深度嵌入开发流程，并且开始意识到依赖单一平台的风险。**

## 为什么大多数人会错过它

主流观点一："封号是个案，概率很低，不需要专门解决。"

错。**问题不是概率，是后果的严重性。** 就像你买保险不是因为火灾概率高，而是因为火灾的代价太大。对于重度用户，一次封号的损失可能是几周的工作量。

主流观点二："用开源工具就没事了，比如 Continue.dev 或者 Ollama。"

也对也不对。开源工具确实没有封号问题，但迁移成本依然存在。你的配置、prompt、项目结构怎么从 Claude Code 迁移到 Continue.dev？目前没有工具帮你做这件事。

主流观点三："直接写个脚本把文件拷出来就行了。"

理论上可以。但实际上 Claude Code 的数据散落在好几个地方：`.claude` 目录、项目配置、终端历史、缓存文件。没有标准格式，没有统一导出接口。你需要逆向工程它的存储结构。

**这就是机会所在：不是解决封号问题，而是解决迁移成本问题。**

## 如果是我，我会怎么做

### 第一天：验证需求

不用写代码。做三件事：

1. **在 HN 那个帖子下面回复**："我做了个工具，可以一键导出 Claude Code 配置和 prompt 历史，导入到 Cursor 或 Continue.dev。有人需要吗？" 附上 Google Form 链接，问三个问题：
   - 你用什么 AI 编码工具？
   - 你积累了多少 prompt / 配置？
   - 如果有一个工具能一键迁移，你愿意付多少钱？（$9 / $19 / $39）

2. **在 r/ClaudeAI 和 r/ChatGPTCoding 发同样的帖子**

3. **手动分析 Claude Code 的数据结构**：看一下 `.claude` 目录里有什么，终端历史存在哪，有没有导出 API。

**7 天验证成功标准**：Google Form 收到 >50 份提交，其中 >20 人选择了 $19 或以上，并且有人在评论区说"这个我真的需要"。

### MVP 方案

不需要做 App。一个 Markdown 文档 + 一个 shell 脚本就够了：

1. **一个 README**：解释这个工具做什么、怎么用、支持哪些工具之间的迁移
2. **一个 shell script**：扫描 `.claude` 目录和终端历史，打包成 JSON，输出迁移报告
3. **一个简单的 landing page**：用 Carrd 或 Vercel 搭，标题"AI Coding Tool Account Backup & Migration"，定价 $19 一次性，附上 Buy Me a Coffee 或 Gumroad 链接

**技术栈**：Shell script + JSON 格式 + Markdown 文档。2 小时能做完。

### 为什么我会成功

- **不需要实时体验**：脚本跑完就能看到结果，不是需要云端服务的产品
- **一次性付费**：用户没有持续订阅的心理负担
- **自然分发**：每次有封号事件发生，就有人在相关帖子里问"怎么迁移"，这就是你的获客渠道
- **有定价锚**：$19 对比迁移失败损失的几天工作量，太便宜了

### 失败条件

这个判断在什么情况下是错的？

1. **如果 Anthropic 明天推出官方导出功能**。可能性存在，但考虑到他们连申诉通道都没做好，导出功能优先级不会高。

2. **如果用户觉得"重头开始配置"的成本其实很低**。对于轻度用户确实如此，但重度用户（每天用 AI 写代码 >4 小时的人）配置积累是巨大的。

3. **如果 Claude Code 的数据结构太封闭，无法逆向**。目前看 `.claude` 目录是公开的 JSON 格式，但未来可能加密或改变结构。
