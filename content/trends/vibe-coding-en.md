## What is it

Vibe Coding is a new approach to software development where indie developers use AI tools like GitHub Copilot, Codex, and other language models to rapidly build functional products. Instead of writing every line of code manually, developers describe what they want in plain English, let the AI generate the code, and then tweak the output. The "vibe" part comes from the flow state — you focus on the product's feel and functionality rather than syntax and boilerplate. For indie developers, this means you can go from idea to prototype in hours instead of days. It’s not about replacing skill; it’s about accelerating execution.

## Why now

Vibe Coding is emerging now because AI coding assistants have crossed a quality threshold. Models like GPT-4 and Codex can generate correct, context-aware code for common tasks. Meanwhile, indie developers face pressure to ship faster with smaller teams. The combination of affordable API costs, improved model accuracy, and a culture of rapid prototyping has made "just prompt it" a viable workflow. Also, developer fatigue with boilerplate and configuration has created demand for tools that let you stay in the creative zone. The timing aligns with a broader shift toward AI-augmented development across the industry.

## Who's behind it

The practice is driven primarily by individual indie developers and small teams sharing their workflows on platforms like GitHub and V2EX. No single company owns Vibe Coding — it’s an emergent behavior. However, OpenAI (Codex), GitHub (Copilot), and Anthropic (Claude) provide the core AI tools. Early adopters include solo founders building SaaS tools, side-project hackers, and developers on V2EX who document their experiments. These communities are the real drivers, sharing prompts, patterns, and cautionary tales. The trend is grassroots, not corporate.
