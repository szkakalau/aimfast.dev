## What is it（这是什么）

Local-First File Sharing 指不依赖云服务器、仅通过局域网（Wi-Fi 或蓝牙）在设备间直接传输文件的工具。用户无需安装客户端、无需联网，只需打开浏览器或轻量应用即可互传，典型如飞投（Feitou）和 WebDrop。它强调零配置、跨平台兼容和隐私安全，适合临时文件交换场景。

## Why now（为什么现在出现）

这一趋势出现的关键驱动力是用户对隐私和便捷性的双重需求爆发。云存储频繁爆出数据泄露事件，而远程办公和跨设备协作的普及让局域网传输成为刚需。同时，WebRTC 等浏览器端点对点技术成熟，使零安装成为可能。独立开发者社区在 w2solo 和 v2ex 上开始讨论这类轻量工具，反映出对传统网盘“重、慢、不透明”的厌倦。

## Who's behind it（谁在推动）

目前主要由独立开发者和开源社区推动。飞投（Feitou）是国内一款受欢迎的局域网文件共享工具；WebDrop 是开源项目，提供基于浏览器的点对点传输方案。此外，v2ex 和 w2solo 上的技术博主和早期用户通过技术分享和口碑传播加速了概念普及。暂无大型公司主导，这为独立开发者留出了空间。

## Market signals（市场信号）

Local-First File Sharing 在技术社区中呈现早期萌芽状态。从 2 个信源（w2solo、v2ex）获得 2 次提及，讨论集中在工具对比和隐私优势上。当前阶段为“nascent”，热度低但话题明确，尚未进入主流视野。跨平台提及模式表明，用户主要关注 Windows、macOS、iOS 和 Android 之间的互通性。

## Commercial opportunities（商业化机会）

独立开发者可围绕 Local-First File Sharing 建立以下产品：
1. **企业内网文件共享工具**：面向团队提供安全、零安装的局域网文件分发服务，适合无互联网的办公环境。
2. **跨平台桌面/移动应用**：集成增强功能（如断点续传、文件预览），通过付费授权或高级功能变现。
3. **嵌入式 SDK**：为其他应用提供局域网传输能力，按调用次数或定制化服务收费。

## Related terms（相关趋势）

- **WebRTC**：Local-First File Sharing 的关键底层技术，支持浏览器端点对点通信。
- **Zero-Install Software**：强调无需安装即可使用的软件理念，与 Local-First 的零配置特征高度关联。
- **Edge Computing**：将计算和存储推向网络边缘，与本地优先的数据处理逻辑一致。

## Product ideas（产品创意）

1. **LanDrop**  
   一款基于浏览器的局域网文件共享工具，支持拖拽传输和二维码扫描，零安装、零配置。  
   现在做是对的时机：用户对云盘隐私担忧加剧，且 WebRTC 技术已稳定，独立开发者可快速上线并占据早期用户心智。

2. **FeiShare Pro**  
   面向团队的局域网文件共享桌面应用，增加权限管理、传输日志和离线缓存功能。  
   现在做是对的时机：远程办公常态化，团队需要安全、高效的内部文件交换方案，而现有工具功能单一，付费意愿明确。

3. **LocalSync SDK**  
   为移动应用开发者提供的局域网传输 SDK，集成文件、图片和文本的快速同步能力。  
   现在做是对的时机：越来越多应用需要本地协同功能，但自研成本高，SDK 模式可降低开发者门槛，抢占生态位。
