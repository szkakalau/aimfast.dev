## What is it

Orzma is a terminal emulator that renders webviews directly inside the terminal window. Think of it as a command line that can also display full HTML, CSS, and JavaScript content inline. Instead of switching between a terminal and a browser, you can run a web app or a dashboard right next to your shell commands. For indie developers, this means you can build rich, interactive UIs for CLI tools without leaving the terminal environment. It’s not just a fancy terminal—it’s a hybrid that bridges the gap between command-line utilities and modern web interfaces, making it easier to create tools that feel both powerful and visually polished.

## Why now

The timing for orzma makes sense because developers are increasingly tired of context switching. We live in a world of Slack, browsers, terminals, and IDEs—all competing for screen real estate. At the same time, web technologies (HTML, CSS, JS) have become the lingua franca for UI development. Terminal emulators have been stagnant for years, mostly just rendering text faster. Orzma emerges as a response to the need for richer, more visual CLI experiences without abandoning the terminal’s speed and scriptability. It also aligns with the rise of tools like tmux and kitty, which already push terminal boundaries, but none have gone this far into webview territory.
