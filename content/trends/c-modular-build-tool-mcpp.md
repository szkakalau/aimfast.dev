## What is it（这是什么）

mcpp 是一款专为 C++ 模块化设计的构建工具。它解决了传统构建系统（如 Make、CMake）对 C++20 模块支持不佳的问题，能高效处理模块依赖、编译顺序和跨平台编译。对独立开发者而言，mcpp 意味着更快的编译速度和更清晰的模块化管理，尤其是在 Linux 上交叉编译 Windows 可执行文件时，能大幅简化流程。

## Why now（为什么现在出现）

C++20 模块化特性发布后，主流编译器（如 GCC 16、Clang）逐步完善支持，但构建工具生态滞后，开发者常面临模块依赖解析混乱、跨平台编译困难等问题。mcpp 的出现正是为了填补这一空白：它新增的 gcc16/mingw 支持，允许开发者在 Linux 上直接生成 Windows 可执行文件，解决了独立开发者和小团队在混合平台开发中的关键痛点。此外，跨平台开发需求增长（如游戏、工具链）也推动了这一趋势。

## Who's behind it（谁在推动）

目前，mcpp 主要由开源社区中的几位核心开发者维护（信源来自 v2ex，表明社区参与度高）。其背后没有大型公司支持，更多是独立开发者或小团队针对 C++ 模块化构建痛点的自发贡献。社区讨论集中在 v2ex 等技术论坛，表明早期采用者主要是对 C++ 模块化有强需求的开发者。
