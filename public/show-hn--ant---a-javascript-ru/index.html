你的项目准备好迁移到 Ant 了吗？

30 秒内分析你的 Node.js 项目，获得一份免费的 Ant 兼容性报告。

Why Ant Compatibility Matters Right Now

上周 Hacker News 上关于 Ant runtime 的讨论获得了 314 个赞和 144 条评论——这是自 Bun 发布以来社区对“新 JavaScript runtime”最强烈的信号。但几乎所有评论都指向同一个问题：“我的 Node.js 项目能直接跑在 Ant 上吗？” 现有的迁移指南要么是通用原则，要么只针对 Bun 生态，没有人为 Ant 做自动化的兼容性诊断。

更关键的是，Ant 的 npm 兼容层仍处于早期阶段，许多常用依赖（比如 Prisma、Puppeteer、node:crypto 的同步写法）可能无法正常工作。手动检查每个依赖的兼容性需要数小时，而且很容易遗漏。开发者需要的是一个工具，能在 30 秒内告诉他们：“你的项目离 Ant 还有多远，具体需要改什么。”

现在正是最好的时机：Ant 的 GitHub 仓库在过去两周内收到了 200+ 个 issue，其中 40% 与兼容性相关。社区正在寻找解决方案，而我们的工具能直接填补这个空白。越早提供诊断能力，就越能成为 Ant 生态中“迁移”这个心智节点的默认选择。

How It Works

1

上传 package.json 或运行 CLI

直接把你的 package.json 拖拽到页面，或者下载我们的一行命令 CLI 工具，在项目根目录运行 `bash check_ant_compat.sh`。两种方式都会立即开始分析你的依赖和 API 使用情况。

2

获得兼容性报告

30 秒内，你会看到一份结构化的报告：哪些依赖在 Ant 的 npm 兼容层下已知工作、哪些可能有问题、哪些完全不兼容。同时扫描出使用了 Node.js 特有 API（如 `fs.promises` 旧写法、`crypto.randomBytes` 同步版本）的代码行。

3

执行迁移清单

报告附带一个可执行的步骤清单，告诉你“替换 `crypto.randomBytes` 为 Ant 原生 API”或“升级 `express` 到 5.x 版本以获得兼容性”。基于 Ant 官方 benchmark，还会估算迁移后可能的性能提升。

What You Get

零配置分析

无需注册、无需安装。上传 package.json 或运行一行命令，30 秒内获得完整的兼容性报告。支持 monorepo 和私有包，所有分析在浏览器或本地完成，代码不会上传到第三方服务器。

不只是标记“不兼容”——每条问题都附带具体的修改方案，包括代码示例和链接到 Ant 官方 API 文档。迁移清单按优先级排序，从“必须修改”到“可选优化”，让你能按部就班地完成迁移。
