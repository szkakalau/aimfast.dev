# AimFast.Dev v3 — 从第一性原理大改计划

> 基于双轨分析 + 用户反馈修正的全面重构方案
> 日期：2026-07-04 | 版本：v2.0（修正版）

## 0. 修正声明

v1.0 计划把"竞品监控"从产品定位中删除了——这是错的。

正确理解：

```
产品 = 发现引擎（Discovery） + 监控引擎（Monitoring） + 决策引擎（Decision）

发现引擎：系统从市场噪音中发现你应该关注的机会（你不需要输入任何东西）
监控引擎：你指定对手/话题，系统每天跨源追踪并告诉你该做什么（你需要输入）
决策引擎：对两个引擎的输出进行压缩 → 你每天只需要做几个决定
```

三者不是互斥的，是互补的。**没有监控引擎，发现引擎就是在黑屋子里扔飞镖。**

## 1. 第一性原理：回到本质

### 1.1 这个产品真正的本质

> **AI 时代的"市场感知 + 竞品追踪 + 决策辅助"系统**

它解决的是：
> 创业者无法实时理解"市场正在如何变化" + "对手在做什么" + "我该做什么"

### 1.2 它不是什么

- ❌ RSS 工具
- ❌ 新闻聚合
- ❌ 纯 AI 总结
- ❌ BI 仪表盘

### 1.3 它是什么（三层定义）

| 层次 | 定义 | 类比 |
|------|------|------|
| **感知层** | 从市场噪音中识别信号 | Bloomberg 的市场扫描 |
| **追踪层** | 对你关心的对手/话题持续追踪 | Bloomberg 的 Watchlist |
| **决策层** | 把信号和追踪结果压缩为行动建议 | Bloomberg 的分析师报告 |

### 1.4 两条信息流

```
                        ┌──────────────────────┐
                        │     信号源（6-8个）     │
                        └──────┬───────────────┘
                               │
               ┌───────────────┴───────────────┐
               ▼                               ▼
    ┌──────────────────────┐     ┌──────────────────────┐
    │  流A：发现引擎         │     │  流B：监控引擎          │
    │  （被动扫描）           │     │  （主动追踪）           │
    │                      │     │                      │
    │  系统自动扫描全市场     │     │  用户定义追踪目标       │
    │  发现新兴机会/趋势     │     │  持续追踪对手/话题      │
    │  输出：≤3 个推荐信号   │     │  输出：每个目标的动态    │
    └──────────┬───────────┘     └──────────┬───────────┘
               │                            │
               └──────────┬─────────────────┘
                          ▼
              ┌──────────────────────┐
              │  决策引擎（Decision）  │
              │                      │
              │  对 A + B 的输出压缩   │
              │  每条信号/动态配反方   │
              │  输出：每天几个决策     │
              │  [关注] [忽略] [行动]  │
              └──────────────────────┘
```

### 2.1 用户故事

```
我输入 "polymarket whale" 作为追踪目标。
系统每天在所有信号源中搜索与它相关的内容。

┌─────────────────────────────────────────────┐
│ 🎯 竞品动态：Polymarket Whale                  │
│                                             │
│ 📊 本周提及：12 次（↑3）                      │
│                                             │
│ 💬 值得关注的评论：                            │
│ ┌─────────────────────────────────────────┐ │
│ │ HN: "Polymarket whale just moved $2M     │ │
│ │      to a new market..."                │ │
│ │ → 翻译：有大户在做某方向                      │ │
│ │ → 对你有用：这可能预示某个市场的热度            │ │
│ └─────────────────────────────────────────┘ │
│ ┌─────────────────────────────────────────┐ │
│ │ Reddit: "Anyone else notice the whale    │ │
│ │         activity on Polymarket?"         │ │
│ │ → 翻译：社区注意到异常交易行为               │ │
│ │ → 对你有用：如果做预测市场工具可以抓这个痛点    │ │
│ └─────────────────────────────────────────┘ │
│                                             │
│ 🤔 本周无关噪音（已帮你过滤）：                  │
│ - 3 条仅提及名称无实质内容的帖子                │

│ [ ] 忽略（现阶段不相关）                        │
└─────────────────────────────────────────────┘
```

### 2.2 追踪目标的类型

用户可以追踪多种类型的"目标"：

| 类型 | 示例 | 追踪什么 |
|------|------|---------|
| **竞品公司/产品** | "polymarket whale"、"Cursor"、"Vercel v0" | 动态、用户反馈、技术变化、融资/增长 |
| **人物** | "Sam Altman"、"Pieter Levels" | 言论、方向变化、新项目 |
| **话题/赛道** | "AI prediction market"、"agent framework" | 讨论趋势、新进入者、痛点变化 |
| **技术栈** | "LangChain"、"Next.js 16" | 版本更新、社区反馈、替代方案兴起 |
| **平台/渠道** | "Product Hunt launch strategy" | 策略变化、新玩法 |

### 2.3 追踪目标管理

```
Dashboard 上有一个「+ 添加追踪」入口：

输入：polymarket whale
类型：[竞品] [人物] [话题] [技术] [平台]
备注：（可选）为什么追踪这个

添加后：
- 每天的采集管线会自动过滤与这个目标相关的信号
- 过滤维度：名称匹配 + 语义关联 + 相关话题
- 追踪状态保持跨天持久化
```

### 2.4 竞品情报的处理规则

对每个追踪目标，每天处理：

```
原始提及（所有源）
    │
    ▼
去噪（过滤：纯转帖、无实质内容、无策略价值）
    │
    ▼
分类：
  ├─ 值得关注的评论（有洞察价值）→ 翻译 + 对你有什么用
  ├─ 纯信息（知道即可）→ 一句话摘要
  └─ 噪音（已过滤，仅在底部计数提示）
    │
    ▼
趋势判断：本周提及量 ↑/↓/→
    │
    ▼
行动建议：基于竞品动态，你应该做什么？
```

```

```

```
采集 → 去重 → 聚类 → 打分
  │

  │
  └─→ 合并输出 → 日报 + Dashboard JSON
```

### 5.3 LLM Prompt 新增维度

在日报生成/信号处理的 prompt 中，新增一个处理阶段：

```
## 竞品情报处理（新增）

对于用户追踪的每个目标，从今日全量信号中提取相关提及。
对每条提及进行三层处理：

1. 去噪
   - 仅提及名称但无实质内容 → 计入噪音计数，不展示
   - 纯转帖/无评论/无上下文 → 同上
   - 纯价格/涨跌讨论 → 酌情保留（如果是竞品产品，涨跌可能是信号）

2. 有价值内容的处理
   - 翻译：把原帖翻译成中文
   - 提炼：这对竞品意味着什么
   - 归因：这对你（追踪者）有什么可行动的
   - 输出格式：

   💬 [来源]: "[原文关键句]"
      → 什么意思：[一句话白话翻译]
      → 对竞品的影响：[竞品可能的方向变化]
      → 对你有用：[你可以做什么]

3. 聚合趋势
   - 本周提及总量 vs 上周
   - 情感倾向（正面/负面/中性）
   - 是否有新进入者/新变化
```

## 7. 竞品监控的核心设计原则

### 7.1 你要的不是"搜索"，是"情报"

```
❌ 搜索：在 11 个源中搜索 "polymarket whale" → 返回所有含这个名字的帖子
✅ 情报：告诉我这些帖子里哪些对决策有用，哪些是噪音，我该做什么
```

关键差异：**情报 = 过滤 + 翻译 + 归因 + 行动**

### 7.2 噪音过滤 > 信号收集

一个追踪目标每天可能被提及 20 次，但只有 2-3 条真正有用。**系统必须做第一层过滤**，不能把所有提及都原样展示。

### 7.3 "对你有什么用"是必填项

竞品动态不是新闻报道。每条有洞察价值的提及，必须回答：
- 这对竞品意味着什么？
- **这对你有什么可行动的？**

### 7.4 历史追踪 > 单日快照

不只是看"今天有什么"，更要看：
- 本周 vs 上周提及量
- 情感趋势变化
- 是否有新的讨论角度出现
