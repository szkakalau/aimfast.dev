# 机会狙击系统（Opportunity Sniper）

> 独立开发者的零成本试错武器 — 每日从全球信号源锁定高确定性机会，1小时出landing page，7天见生死
>
> 版本：v0.3 | 日期：2026-05-23 | 作者：大川

---

## 1. 产品概述

### 1.1 一句话定义
每日从全球+中国信号源中锁定高确定性机会，1小时出landing page，7天验证见生死。

### 1.2 核心理念：狙击，不是扫射
不缺想法，缺的是**快速验证和果断放弃的纪律**。

这套系统不是给你100个方向让你选，而是：
- 每天锁定1个最强机会（3个交叉信号验证）
- 1小时做出landing page，丢到痛点现场
- 7天看数据，有流量注册→做产品，没有→果断放弃

**狙击手的素质：一击不中，立刻撤退，不恋战。**

### 1.3 使用场景（自用）
- 每天早上看一眼今天有什么机会信号
- 信号够强 → 1小时做个landing page丢出去
- landing page有流量/注册/咨询 → 这才是真正该投入的项目
- 没流量 → 果断放弃，成本只有1小时

### 1.4 商用预留
当前自用，但架构上预留以下扩展点：
- 多用户：信号采集和打分是无状态的，加用户画像过滤层即可
- 付费版：Pro内容（竞品深度分析、SEO数据、landing page模板）独立模块
- API：信号数据标准化输出，方便对接其他工具
- 品牌化：公众号/即刻分发模块独立，随时可开

### 2.3 Landing Page语言规则
- **海外需求** → 纯英文landing page（面向全球用户，英文流量更大、付费意愿更高）

```markdown
### 域名策略：一域千弹

**一个域名，子目录部署所有landing page。不要为每个项目买新域名。**

```
aimfast.dev/（主域名，选一个短且通用的）
├── /source-credit/        ← 今日机会的landing page
├── /ai-reply-fatigue/     ← 上周验证中的
├── /cloud-kill-switch/    ← 已放弃（保留，不删）
├── /agent-test-trust/     ← 已放弃
└── /（首页，放当前最强机会 + 历史验证成果）
```

**为什么不用子域名（sourcecredit.sniper.dev）？**
- 子目录所有页面共享一个域名的权重，页面越多权重越高
- 子域名在SEO上是独立站点，权重不共享
- 验证阶段靠社区投放不走SEO，但老页面会慢慢积累长尾流量——这些流量全部反哺 aimfast.dev

**为什么新项目不买新域名？**
- 一个域名 $10-15/年，做10个验证 = $100-150/年，大部分会放弃
- 一个域名 $10-15/年，无限个子目录 = 永远这个价
- 新域名零权重，老域名的子目录天然继承权重
- 放弃的项目页面不删：成本低、积累SEO、偶尔还有长尾流量进来

**域名选择原则：**
- 短（6字母以内最佳）
- 通用（不绑定某个具体方向）
- 好记（别人在社区看到链接愿意点）
- 推荐后缀：`.dev` / `.app` / `.io`（开发者信任感强）

**已选域名：aimfast.dev**
- aim = 瞄准/狙击，fast = 快速验证，完美匹配"机会狙击系统"
- 4+4字母，短且好记
- 海外开发者看到会觉得酷

**流量最大化策略：**
| 做法 | 效果 |
|------|------|
| 每个landing page都是子目录 | 所有页面权重汇入一个域名 |
| 放弃的页面不删，加一句"实验已结束" | 长尾搜索流量继续流入主站 |
| 主站首页展示最新机会+已验证成功案例 | 新访客看到社会证明，转化率提高 |
| 每个页面底部互相链接 | 内链网络，提升整体权重 |
| 1个域名积累3个月后 | 域名权重起来了，新页面更快被收录 |

### 搭建方式
- 推荐：Vercel（免费，自动部署，子目录天然支持）
- 框架：Next.js或纯HTML（选最快的）
- 部署：push到GitHub → Vercel自动部署，0配置

### 验证指标
- 7天目标：[XX uv / XX 注册 / XX 邮件]
- 达标 → 做产品
- 未达标 → 放弃或调整方向

### 流量策略（7天验证周期）

**核心原则：Landing Page不是等流量来的，是你带着解决方案去痛点现场找用户。**

#### 第一阶段：社区投放（Day 1-7，零成本验证）
信号源本身就是流量入口。每个机会的信号来自哪里，就去哪里发帖：

- 海外机会 → HN/Reddit/DEV Community 相关帖子下回复或发Show HN
- 国内机会 → V2EX/即刻/小红书 相关话题下发帖

这不是硬推广告，是**问题现场摆解决方案**。讨论这个问题的人 = 你的目标用户。

| 步骤 | 操作 | 话术要点 |
|------|------|---------|
| 1 | 找到信号来源帖 | 读完原帖和热门评论 |
| 2 | 写有价值的回复 | 先认同痛点，再说"我做了一个XX，可以帮你YY" |
| 3 | 附landing page链接 | 不要直接丢链接，要给上下文 |

#### 第二阶段：验证判断（Day 7）
- 社区帖子带来 > 100uv 且有注册/咨询 → 值得投入做产品
- 社区帖子带来 < 30uv → 问题不痛或方案不对，放弃
- 30-100uv 但零注册 → 流量对但转化差，调整landing page或放弃

#### 第三阶段：只有验证通过才投入长期流量（Day 7+）
验证通过的项目才值得做SEO/内容/广告：
- SEO长尾词：针对具体痛点关键词做内容页
- 微内容引流：写相关教程/文章发布到Medium/知乎/DEV
- Product Hunt正式发布
- 付费广告（如果LTV测算成立）

#### 反面提醒
- ❌ 新landing page不做SEO——零权重域名，做不了
- ❌ 不花钱买广告验证——花钱买到的信号不真实
- ❌ 不在无关社区硬推——转化率为零还伤品牌
```

## 3. 信号采集系统

### 3.1 信号源

#### 海外信号源（中文解读）

| 信号源 | 采集内容 | 采集方式 | 优先级 |
|--------|---------|---------|--------|
| Hacker News | Top stories + 评论数 + Show HN | Algolia API | P0 |
| GitHub Trending | 热门repo + star增长 | 爬虫 | P0 |
| Product Hunt | 当日新品 + upvote | API | P1 |
| HuggingFace | 热门模型 + 下载量 | API | P1 |
| Google Trends | 搜索词暴涨/降温 | pytrends | P0 |
| Reddit | 热帖 + 子版块 | RSS | P1 |

#### 中国信号源

| 信号源 | 采集内容 | 采集方式 | 优先级 |
|--------|---------|---------|--------|
| V2EX | 创意/分享区热帖 | API | P0 |
| 即刻 | 创业/副业圈热门 | mobile_use | P0 |
| 小红书 | 创业/副业爆款 | 爆款检测skill | P0 |
| TikTok热销品 | 海外热销品+GMV | 已有定时任务 | P1 |
| 知乎 | 创业热问 | 搜索 | P2 |

### 3.2 采集规范
- 每条信号：来源URL + 标题 + 讨论量 + 采集时间
- 同一事件跨平台 → 合并为一条，标注多平台验证
- 存储：`./BuilderRadar/raw/YYYY-MM-DD/{source}.json`
- 原始数据保留7天

### 3.3 信号打分

```
Score = cross_platform×3 + volume×2 + freshness×2 + actionability×1
```

| 维度 | 评分规则 |
|------|---------|
| cross_platform | 1平台=1分, 2平台=3分, 3平台+=5分 |
| volume | <50=1分, 50-200=2分, 200-500=3分, 500+=5分 |
| freshness | 今天=5分, 昨天=3分, 持续3天+=1分 |
| actionability | 能推导出具体产品=5分, 方向模糊=3分, 纯抱怨=1分 |

**触发Landing Page的阈值：Score ≥ 12 且 cross_platform ≥ 3**

## 6. 狙击循环

```
信号锁定 → 1小时出Landing Page → 扔到痛点现场 → 7天见生死
   ↑                                              │
   │                                    ┌─────────┴──────────┐
   │                                    │                    │
   │                                 有流量注册            没流量
   │                                    │                    │
   │                               做产品+建流量         果断放弃
   │                                    │                    │
   │                              自然流量飞轮          1小时成本，归档经验
   │                                    │                    │
   └────────────────────────────────────┘                    │
                          ↑                                 │
                          └─── 经验反哺信号判断 ─────────────┘
```

**狙击手和扫射的区别：扫射打100发中1发，狙击1发不中换位置。**

每个landing page只花1小时，7天看数据：
- 有流量有注册 → 值得做，投入建产品+SEO+内容
- 没流量 → 放弃，成本只有1小时
- 放弃的案例积累成经验库，反哺未来的信号判断
